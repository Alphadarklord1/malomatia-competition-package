# Code of Conduct

## Our Standard

Contributors are expected to keep discussion and collaboration respectful,
direct, and constructive.

Examples of expected behavior:

- giving actionable technical feedback
- respecting different experience levels
- focusing criticism on code and decisions, not people
- documenting assumptions clearly
- helping keep the repository safe for collaboration

Examples of unacceptable behavior:

- harassment or intimidation
- personal attacks
- discriminatory language
- deliberately disruptive behavior
- sharing private credentials or sensitive data in issues or pull requests

## Enforcement

Project maintainers may edit, remove, or reject contributions that violate this
code of conduct.

## Reporting

If a collaboration issue needs maintainer attention, report it privately to the
repository owner through GitHub.
